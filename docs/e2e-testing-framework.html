ReportsFW End-to-End Testing Framework

This document defines the E2E automation framework introduced for ReportsFW. It is designed for maintainable business-flow testing with reusable page objects and deterministic test data.

1. Stack and Runner

  • pytest as test runner and fixture orchestrator
  • selenium for browser automation
  • webdriver-manager for ChromeDriver lifecycle
  • Headless Chrome default, configurable through environment variables

2. Framework Layout

  • tests/e2e/conftest.py: browser + wait fixtures
  • tests/e2e/pages/base_page.py: common interaction helpers
  • tests/e2e/pages/*_page.py: feature-specific page objects
  • tests/e2e/tests/test_dashboard_smoke.py: smoke validation
  • tests/e2e/tests/test_business_flows.py: full lifecycle scenario

3. Test Controls

VariableDefaultPurpose
E2E_BASE_URLhttp://localhost:3000Application host under test
E2E_TIMEOUT_SECONDS15Explicit wait timeout
E2E_HEADLESStrueHeadless/non-headless browser mode

4. Execution Guide

  1. Start the app: npm run dev
  2. Install E2E dependencies: python -m pip install -r tests/e2e/requirements-e2e.txt
  3. Run all E2E tests: python -m pytest
  4. Run a specific test file: python -m pytest tests/e2e/tests/test_business_flows.py

5. Design Principles

  • Page Object pattern for reduced selector duplication
  • Business-level assertions (text and flow outcomes) over brittle implementation checks
  • Timestamp-based unique test data to avoid collisions
  • Single browser session per run for speed, explicit waits for stability
