ReportsFW Business Use Cases

This document captures business use cases covered by the E2E framework, including primary actors, expected outcomes, and traceability to automated tests.

1. Primary Actors

Finance Operations Manager
Equity Plan Administrator
HR Compensation Analyst
Executive Stakeholder

2. Core Business Use Cases

ID Use Case Goal Main Steps Success Criteria
UC-01 Review Portfolio Dashboard View latest portfolio and risk overview. Open dashboard, refresh data, view chart and summary sections. Key dashboard sections are visible and data loads without crash.
UC-02 Create Company Profile Add a new legal entity for reporting and plan assignment. Navigate to companies page, enter company details, submit form. Company appears in list and can be selected for downstream entities.
UC-03 Register Employee Associate an employee with a company and compensation profile. Open employees page, select company, submit employee details. Employee appears in directory with correct identity fields.
UC-04 Create Stock Plan Define plan structure and contribution settings for a company. Open stock plans page, select company, enter plan data, submit. Plan is persisted and displayed in plan catalog and dashboard context.
UC-05 Validate End-to-End Traceability Confirm newly created records surface on the dashboard. Execute UC-02, UC-03, UC-04 then revisit dashboard. New company and plan names are visible on dashboard-driven views.

3. Test Traceability Matrix

Use Case Automated Test Coverage Notes
UC-01 test_dashboard_renders_key_sections Validates dashboard render, refresh action, and critical section visibility.
UC-02, UC-03, UC-04, UC-05 test_end_to_end_business_flow Exercises create flows for company, employee, stock plan, then verifies dashboard propagation.
