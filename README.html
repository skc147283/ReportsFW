Project Snapshot

  • Next.js 16 and TypeScript dashboard for stock-plan reporting.
  • API routes in src/app/api/* power dashboard and CRUD workflows.
  • Primary data source is PostgreSQL with Oracle and demo fallback options.
  • E2E automation uses pytest and Selenium page objects.
Next.js 16 TypeScript PostgreSQL Oracle (optional) pytest + Selenium

Architecture

The runtime follows a layered model where App Router pages call API routes, and API routes delegate all persistence operations to a single data-access layer. 

Browser UI
-> App Router pages (dashboard + managers)
-> API routes (/api/dashboard, /api/companies, /api/employees, /api/stock-plans)
-> Data access module (src/lib/oracle.ts)
-> PostgreSQL (preferred) / Oracle (secondary) / in-memory demo fallback

Core Responsibilities

  • src/app/* and src/components/*: UI rendering and user interactions.
  • src/app/api/*/route.ts: request validation and API orchestration.
  • src/lib/oracle.ts: DB connection, reads/writes, schema seed, fallback behavior.
  • db/schema.sql: relational model definition and seed examples.

Testing Framework

The E2E suite validates user-critical flows through browser automation with reusable page objects and stable explicit waits.

Structure

  • tests/e2e/conftest.py: browser and wait fixtures.
  • tests/e2e/pages/*.py: page object classes.
  • tests/e2e/tests/test_dashboard_smoke.py: smoke coverage.
  • tests/e2e/tests/test_business_flows.py: full create-and-verify flow.

Run Commands

  • python3 -m pip install -r tests/e2e/requirements-e2e.txt
  • python3 -m pytest

Configurable with E2E_BASE_URL, E2E_HEADLESS, and E2E_TIMEOUT_SECONDS.
